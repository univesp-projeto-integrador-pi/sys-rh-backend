
# 🚀 Guia de Sobrevivência do Programador Iniciante

Este guia foi criado para explicar os conceitos fundamentais da tecnologia de um jeito simples, como se estivéssemos conversando no intervalo da escola.

-----

## 🌐 1. O que é uma Requisição HTTP?

Imagine que você está em um restaurante:

  * **Você** é o "Cliente" (Navegador).
  * **O Garçom** é o "HTTP" (O mensageiro).
  * **A Cozinha** é o "Servidor" (Onde o site fica guardado).

Quando você pede um hambúrguer, você faz uma **Requisição**. O garçom leva seu pedido e traz o lanche (**Resposta**). Na internet, quando você clica em um link, seu navegador pede os dados e o HTTP traz a página para a sua tela.

-----

## 🐙 2. Git e GitHub: Qual a diferença?

  * **Git (A Máquina do Tempo):** É um programa que você instala no seu computador. Ele tira "fotos" (saves) do seu código. Se você fizer uma besteira e apagar tudo, pode usar o Git para voltar no tempo até a última foto que tirou.
  * **GitHub (A Nuvem dos Devs):** É um site onde você guarda as fotos que o Git tirou. É como o Instagram, mas em vez de fotos de comida, você posta códigos para outros verem.
  * **Como se complementam:** Você usa o **Git** para organizar seu trabalho no PC e o **GitHub** para salvar na nuvem e trabalhar em grupo.

-----

## 🌿 3. O que é uma Branch? (O Universo Paralelo)

Imagine que você está desenhando um herói.

  * A **Main** (ou Master) é o seu desenho principal, que está ficando lindo.
  * Você quer testar pintar o herói de verde, mas tem medo de estragar tudo.
  * Você cria uma **Branch** (Ramo). É como se você tirasse uma cópia do desenho para pintar em outra folha.
  * Se o herói verde ficar legal, você junta com o oficial. Se ficar feio, é só amassar a folha da Branch e o seu desenho principal continua perfeito\!

## 🐳 4. Docker e Docker Compose

  * **Docker (A Caixa Mágica):** Sabe quando um jogo funciona no PC do seu amigo, mas não no seu? O Docker resolve isso criando "caixas" (Containers). Dentro da caixa tem tudo o que o programa precisa para rodar. Se funciona na caixa, funciona em qualquer lugar.
  * **Docker Compose (O Maestro):** Se o Docker cuida de uma caixa, o Compose cuida de várias. Ele é o manual que diz: "Ligue a caixa do Banco de Dados e a caixa do Site juntas".

### 🐧 Docker no Linux vs. 🪟 Docker no Windows

  * **No Linux:** O Docker fala a mesma língua do sistema. É super rápido e leve.
  * **No Windows:** O Windows precisa de um tradutor chamado **WSL 2** (um mini Linux escondido). Funciona bem, mas gasta um pouco mais de memória.

## 🔃 5. PR, Pull e Merge

Imagine que você e um amigo estão escrevendo um livro juntos:

  * **Pull (Puxar):** Você baixa no seu PC o que seu amigo escreveu para ter a versão mais nova.
  * **PR - Pull Request (Pedido de Puxada):** Você termina sua parte em uma **Branch** e pede: "Ei, olha o que eu fiz\! Pode colocar no livro oficial?".
  * **Merge (Fundir):** Seu amigo aprovou e agora o Git "cola" a sua Branch dentro da história principal do livro.

## 🟢 6. O que é Node.js?

Antigamente, o JavaScript só funcionava dentro do navegador (Chrome). O **Node.js** tirou o motor do navegador e permitiu que ele rodasse direto no computador. Agora, usamos JS para criar servidores e aplicativos de celular.

-----

## 🔌 7. O que é uma API?

Uma **API** é uma ponte. Se o seu app quer saber o resultado do jogo de futebol, ele não vai no estádio. Ele pergunta para a **API** de esportes: "Quem ganhou?". A API responde e o seu app mostra na tela.
