# 🚀 Guia de Sobrevivência do Programador Iniciante

Este guia foi criado para explicar os conceitos fundamentais da tecnologia de um jeito simples, como se estivéssemos conversando no intervalo da escola.

-----

## 🌐 1. O que é uma Requisição HTTP?

Imagine que você está em um restaurante:

  * **Você** é o "Cliente" (Navegador).
  * O **Garçom** é o "HTTP" (O mensageiro).
  * A **Cozinha** é o "Servidor" (Onde o site fica guardado).

Quando você pede um hambúrguer, você faz uma **Requisição**. O garçom leva seu pedido e traz o lanche (**Resposta**). Na internet, quando você clica em um link, seu navegador pede os dados e o HTTP traz a página para a sua tela.

-----

## 🐙 2. Git e GitHub: Qual a diferença?

  * **Git (A Máquina do Tempo):** É um programa que você instala no seu computador. Ele tira "fotos" (saves) do seu código. Se você fizer uma besteira e apagar tudo, pode usar o Git para voltar no tempo até a última foto que tirou.
  * **GitHub (A Nuvem dos Devs):** É um site onde você guarda as fotos que o Git tirou. É como o Google Drive, mas focado em código, onde outros programadores podem ver seu trabalho.
  * **Como se complementam:** Você usa o **Git** para organizar seu trabalho no PC e o **GitHub** para salvar na nuvem e trabalhar em grupo.

-----

## 🐳 3. Docker e Docker Compose

  * **Docker (A Caixa Mágica):** Sabe quando um jogo funciona no PC do seu amigo, mas não no seu? O Docker resolve isso criando "caixas" (Containers). Dentro da caixa tem tudo o que o programa precisa para rodar. Se a caixa funciona num PC, vai funcionar em qualquer lugar do mundo.
  * **Docker Compose (O Maestro):** Se o Docker cuida de uma caixa, o Compose cuida de várias ao mesmo tempo. Imagine que você tem uma caixa para o Banco de Dados e outra para o Site. O Compose é o manual que diz: "Ligue essas duas caixas juntas e faça elas conversarem".

### 🐧 Docker no Linux vs. 🪟 Docker no Windows

  * **No Linux (O Habitat Natural):** O Docker nasceu para o Linux. Eles falam a mesma língua, por isso ele é super rápido e leve.
  * **No Windows (O Visitante):** O Windows fala uma língua diferente. Para rodar o Docker, ele precisa de um tradutor chamado **WSL 2** (um mini Linux dentro do Windows). Funciona muito bem, mas exige um pouco mais de memória do seu PC.

## 🔃 4. PR, Pull e Merge

Imagine que você e um amigo estão escrevendo um livro juntos:

  * **Pull (Puxar):** Você baixa no seu PC o que seu amigo escreveu para ficar com a versão mais nova.
  * **PR - Pull Request (Pedido de Puxada):** Você termina um capítulo e diz: "Ei, olha o que eu fiz\! Pode colocar no livro oficial?".
  * **Merge (Fundir):** Seu amigo leu, aprovou e agora ele "cola" o seu capítulo no arquivo principal do livro.

-----

## 🟢 5. O que é Node.js?

Antigamente, o JavaScript só funcionava dentro do navegador (como o Chrome). O **Node.js** é como se tivessem tirado o motor do navegador e colocado ele para rodar direto no computador. Agora, podemos usar JavaScript para criar servidores, robôs e muito mais, sem precisar de um site aberto.

-----

## 🔌 6. O que é uma API?

Uma **API** é uma ponte entre dois programas.

  * **Exemplo:** Você cria um app de clima. Em vez de lançar um satélite no espaço, seu app pergunta para a **API** do Google: "Qual a temperatura em SP?". A API responde "25°C" e seu app mostra na tela. É um jeito de sistemas diferentes trocarem informações.
